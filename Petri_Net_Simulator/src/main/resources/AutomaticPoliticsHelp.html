Help about Automatics Politcs Module 

Funcionalidad

Este módulo permitirá ampliar la eficiencia y flexibilidad de modelación para la definición de políticas. Esto es introduciendo la posibilidad de regular las políticas de disparo de manera dinámica, a medida que la red evoluciona, de manera tal que se optimice la utilización de recursos presentes en el sistema.
Este control se realizará introduciendo nuevas modificaciones en la red a través de múltiples autómatas asociados a cada conflicto.

El módulo cuenta con las siguientes funcionalidades:

  • Obtención del vector de costos, definido desde la interfaz de propiedades de cada transición.
  • Ejecución del algoritmo de regulación de políticas propiamente dicho. Obteniendo como resultado, la evolución de las probabilidades de cada conflicto presente en la red.
    Este algoritmo cuenta con dos modos de ejecución:
    • Ejecución con costos simples: n este modo se buscará optimizar el uso de los recursos utilizando sólo los costos de las transiciones asociadas a cada conflicto.
    • Ejecución de costos invariantes: En este modo se buscará optimizar el uso de los recursos teniendo en cuenta el comportamiento global de la red, analizando el costo de los invariantes de transición. Esto será de utilidad para el caso en que se presenten múltiples conflictos dependientes y la solución encontrada por el primer modo no sea la óptima.
    Además, el módulo permitirá al usuario ajustar los distintos parámetros de ejecución explicados a continuación:
    • Número de iteraciones: Define el número de disparos que el algoritmo realizará sobre la red para determinar cuales son las secuencias de disparos más eficientes de acuerdo a los costos especificados.
    • Repeticiones: Define la cantidad de veces que el algoritmo se ejecutará. Esto será de utilidad para identificar de manera más consistente cuales son los patrones de comportamiento de la nueva red controlada.
    • Casilla de verificación 'Modify net': Utilizada para aplicar las modificaciones de la nueva red controlada a la red actual.
  • Mostrar el Gráfico de las probabilidades calculadas.
  • Verificar automáticamente si se trata de una red de Petri con plazas de control (agregadas por el algoritmo de supervisación de interbloqueo).
  • Verificar automáticamente si se trata de una red la cual ya ha sido modificada por el mismo algoritmo para asegurar el correcto funcionamiento del mismo.

Modo de uso

  • Especificar los costos individuales, en el panel de propiedades de cada transición.
  • Especificar en la interfaz del módulo todos los parámetros del algoritmo.
  • Ejecutar el análisis con el botón de 'Run Politic analysis' y observar el gráfico de probabilidades, mediante el botón de 'show plot', para verificar si el comportamiento del sistema es el deseado.
  • Una vez satisfecho con el comportamiento de la nueva red, aplicar las modificaciones habilitando la casilla de 'Modify Net' y ejecutando nuevamente el análisis.
