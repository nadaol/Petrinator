Help about Deadlock Supervisor Module 

Funcionalidad

El siguiente módulo ejecuta un algoritmo capaz de solucionar los problemas de vivacidad de las redes de Petri(RdP), puntualmente las que modelan Sistemas de procesos secuenciales simples con recursos (S3PR).

El módulo cumple las siguientes funciones:

  • Verificar si la red contiene efectivamente estados de Interbloqueo y si se trata de una red del tipo S3PR.
  • Agregar supervisores (Plazas de control) a la red para lograr eliminar el interbloqueo.
  • Agregar y/o eliminar Arcos para corregir potenciales discrepancias con los invariantes de transición originales.

Modo de uso

El módulo cuenta con cuatro botones, dos de ellos son utilizados para el análisis general y otros dos para la modificación de la red en cuestión.
A continuación, se listarán y explicará cada botón en orden de uso para una correcta supervisión.

  1. First Analysis: Este botón de análisis se habilitará únicamente si la red no cuenta con supervisores. El mismo se ocupará de realizar el primer análisis, en donde se verificará la compatibilidad de la red con el algoritmo. Esto es, verificando tanto la presencia de estados de interbloqueos como el cumplimiento de los requerimientos de la clasificación S3PR. Luego, en el caso que se cumplan las dos verificaciones, el módulo calculará los posibles supervisores a agregar y se deberá continuar con la supervisión, en caso contrario se detallarán en pantalla cuales fueron las razones de incompatibilidad de la red con el algoritmo.
  2. Add Supervisor/s y Fix Conflict/s:Estos dos botones serán los encargados de realizar las modificaciones de la red en cuestión.
    • Add Supervisor/s: Se recomienda utilizarlo antes de realizar la corrección de conflictos. Al accionar el botón se deberá indicar de la lista presentada, el id del supervisor a agregar.
      Importante: Cabe aclarar que el mismo no permitirá el agregado de supervisores con un marcado nulo, ya que, estos no corregirían el estado de interbloqueo. De igual manera debido a la naturaleza del algoritmo, se mostrarán como posibles supervisores, ya que si al menos uno se presenta, esto indicará una posible modificación de los invariantes originales. Además, por razones de convergencia, es recomendable controlar el sifón con mayor cantidad de plazas.
    • Fix Conflict/s: Como se mencionó, en caso que en análisis retorne supervisores con marcado nulo, es recomendable primero hacer un arreglo de los conflictos. En caso que algún invariante se haya modificado respecto a los originales, estos serán restablecidos.
  3. Net Analysis (with supervisors): El siguiente botón de análisis se habilitará únicamente si la red cuenta con supervisores agregados. El mismo verificará si la red cuenta con interbloqueo y en este caso calculará los posibles supervisores a agregar y se listarán para continuar con la supervisión.
