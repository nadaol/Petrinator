Help about Deadlock Supervisor Module 

Funcionalidad

El siguiente modulo ejecuta un algoritmo capaz de solucionar los problemas de vivacidad de las redes de Petri(RdP), puntualmente las que modelan Sistemas de procesos secuenciales simples con recursos (S3PR).

El modulo tiene las siguientes funcionalidades:

  • Verificar si la RdP esta en estado de Interbloqueo y si se trata de una red S3PR,
  • Agregar supervisores (Plazas de control) a la RdP para lograr corregir el Interbloqueo.
  • Agregar Arcos a la RdP para corregir problemas con los T-Invariantesde la red sin supervisores

Modo de uso

El modulo cuenta con cuatro botones, cada uno con una funcionalidad y limitado por ciertas verificaciones automáticas, dos botones son para el análisis y otras dos para la modificación de la red, se listaran y explicaran en orden de uso cada botón.

  1. First Analysis: Este botón de análisis se habilita solo y al principio de ejecutar el modulo únicamente si la red no cuenta con supervisores.Realizara el primer análisis de la red donde verificara el Interbloqueo y si es S3PR, solo en caso que las dos verificaciones sean verdaderas,el modulo calculara los posibles supervisores a agregar y habilitara los demás botones, en caso contrario el modulo no tiene mas funcionalidad.
  2. Add Supervisor/s y Fix Conflict/s:Estos dos botones son los encargados de modificar la red para reparar el Interbloqueo.
    • Add Supervisor/s: Recomendable siempre primero, al apretar el botón aparecen una lista con los id de los supervisores para agregar a la red.
      Importante: agregar supervisores con marcado distinto de cero y recomendable el que controle mayor cantidad de plazas, en caso de seleccionar un id de un supervisor con marcado igual a cero saldrá una advertencia y no lo permitirá, ya que la red se rompe en ese caso.
    • Fix Conflict/s: En caso que en análisis retorne supervisores con marcado igual a cero, siempre es recomendable hacer un arreglo de los conflictos (mediante este boton), que corregirá los T-Invariantes de la red original ya que los supervisores agregados los modificaron.
      Importante: si un análisis retorno muchos supervisores y algunos con marcado igual a cero, antes de agregar otro supervisor con marcado es recomendable ejecutar esta opción.
  3. Net Analysis (with supervisors): Este botón de análisis se ejecuta para realizar el análisis cuando una red tiene o se le agrega un supervisor, verifica si la red sigue o no teniendo interbloqueo y en caso que tenga hace el análisis de los supervisores para luego mediante los botones de modificación arreglar la red
